Architecture du Système

Vue d'ensemble

SAMA CONAI est construit sur l'architecture modulaire d'Odoo avec les composants suivants :

Couche Présentation :
  • Interface web responsive
  • API REST pour applications mobiles
  • Tableaux de bord interactifs
Couche Métier :
  • Modèles Odoo (ORM)
  • Workflows automatisés
  • Règles de validation
Couche Données :
  • PostgreSQL (base principale)
  • Système de fichiers (documents)
  • Cache Redis (optionnel)
Intégrations :
  • Services de notification (SMS/Email)
  • Systèmes d'authentification externes
  • APIs gouvernementales

API REST

Authentification

L'API utilise l'authentification par token JWT :

# Obtenir un token
curl -X POST http://votre-serveur/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username": "user", "password": "pass"}'

# Utiliser le token
curl -X GET http://votre-serveur/api/demandes \
  -H "Authorization: Bearer YOUR_TOKEN"
Endpoints Principaux
Méthode Endpoint Description Paramètres
GET /api/demandes Liste des demandes limit, offset, state
POST /api/demandes Créer une demande objet, description, organisme_id
GET /api/demandes/{id} Détail d'une demande -
PUT /api/demandes/{id} Modifier une demande Champs à modifier
POST /api/alertes Créer une alerte type, description, anonyme
GET /api/organismes Liste des organismes -
Exemple d'utilisation
Créer une nouvelle demande :
// JavaScript/Node.js
const response = await fetch('/api/demandes', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer ' + token
  },
  body: JSON.stringify({
    objet: 'Demande d\'information sur le budget',
    description: 'Je souhaite obtenir des informations...',
    organisme_id: 1,
    type_info: 'budget'
  })
});

const demande = await response.json();
Python :
import requests

headers = {
    'Authorization': f'Bearer {token}',
    'Content-Type': 'application/json'
}

data = {
    'objet': 'Demande d\'information sur le budget',
    'description': 'Je souhaite obtenir des informations...',
    'organisme_id': 1,
    'type_info': 'budget'
}

response = requests.post('/api/demandes', json=data, headers=headers)
demande = response.json()

Modèles de Données

Modèle Demande
class Demande(models.Model):
    _name = 'conai.demande'
    _description = 'Demande d\'accès à l\'information'
    _order = 'date_creation desc'

    # Champs principaux
    numero = fields.Char('Numéro', required=True, readonly=True)
    objet = fields.Char('Objet', required=True)
    description = fields.Text('Description détaillée', required=True)
    
    # Relations
    demandeur_id = fields.Many2one('res.partner', 'Demandeur')
    organisme_id = fields.Many2one('conai.organisme', 'Organisme')
    
    # États et dates
    state = fields.Selection([
        ('draft', 'Brouillon'),
        ('submitted', 'Soumise'),
        ('processing', 'En traitement'),
        ('completed', 'Traitée'),
        ('rejected', 'Rejetée')
    ], default='draft')
    
    date_creation = fields.Datetime('Date de création', default=fields.Datetime.now)
    date_limite = fields.Datetime('Date limite de réponse')
    
    # Pièces jointes
    document_ids = fields.One2many('ir.attachment', 'res_id', 'Documents')
Modèle Alerte
class Alerte(models.Model):
    _name = 'conai.alerte'
    _description = 'Alerte citoyenne'

    # Identification
    numero = fields.Char('Numéro', required=True, readonly=True)
    type_alerte = fields.Selection([
        ('corruption', 'Corruption'),
        ('mauvaise_gestion', 'Mauvaise gestion'),
        ('non_transparence', 'Non-transparence'),
        ('autre', 'Autre')
    ], required=True)
    
    # Contenu
    titre = fields.Char('Titre', required=True)
    description = fields.Text('Description', required=True)
    lieu = fields.Char('Lieu')
    
    # Anonymat et confidentialité
    anonyme = fields.Boolean('Signalement anonyme', default=False)
    contact_autorise = fields.Boolean('Contact autorisé', default=True)
    
    # Traitement
    state = fields.Selection([
        ('new', 'Nouvelle'),
        ('investigating', 'En enquête'),
        ('resolved', 'Résolue'),
        ('closed', 'Fermée')
    ], default='new')
    
    priorite = fields.Selection([
        ('low', 'Faible'),
        ('medium', 'Moyenne'),
        ('high', 'Élevée'),
        ('urgent', 'Urgente')
    ], default='medium')

Règles de Sécurité
Exemple de règle d'accès :
<record id="demande_citizen_rule" model="ir.rule">
    <field name="name">Demandes - Accès Citoyen</field>
    <field name="model_id" ref="model_conai_demande"/>
    <field name="domain_force">[('demandeur_id', '=', user.partner_id.id)]</field>
    <field name="groups" eval="[(4, ref('group_conai_citizen'))]"/>
    <field name="perm_read" eval="True"/>
    <field name="perm_write" eval="True"/>
    <field name="perm_create" eval="True"/>
    <field name="perm_unlink" eval="False"/>
</record>
Important : Les alertes anonymes sont chiffrées et stockées séparément pour garantir la confidentialité.

Guide de Développement

Environnement de Développement
Configuration requise :
# Cloner le projet
git clone https://github.com/sama-solutions/sama_conai.git
cd sama_conai

# Installer les dépendances
pip install -r requirements.txt

# Configuration de développement
cp config/dev.conf.example config/dev.conf
# Éditer config/dev.conf avec vos paramètres

# Lancer en mode développement
./odoo-bin -c config/dev.conf --dev=all
Structure recommandée :
  • Utilisez des branches feature pour chaque nouvelle fonctionnalité
  • Respectez les conventions de nommage Odoo
  • Documentez votre code avec des docstrings
  • Écrivez des tests unitaires pour les nouvelles fonctionnalités
Bonnes Pratiques
Modèles :
  • Utilisez des noms de modèles descriptifs avec le préfixe 'conai.'
  • Définissez toujours _description et _order
  • Implémentez les méthodes de validation appropriées
  • Utilisez les contraintes SQL et Python
Vues :
  • Organisez les champs de manière logique
  • Utilisez des groupes pour structurer les formulaires
  • Implémentez des filtres et groupements utiles
  • Ajoutez des boutons d'action contextuels
Contrôleurs :
  • Validez toujours les données d'entrée
  • Gérez les erreurs de manière appropriée
  • Utilisez les décorateurs de sécurité
  • Documentez vos endpoints API

Tests

Tests Unitaires
Exemple de test :
from odoo.tests.common import TransactionCase

class TestDemande(TransactionCase):
    
    def setUp(self):
        super().setUp()
        self.Demande = self.env['conai.demande']
        self.organisme = self.env['conai.organisme'].create({
            'name': 'Ministère Test',
            'code': 'MT'
        })
    
    def test_create_demande(self):
        """Test de création d'une demande"""
        demande = self.Demande.create({
            'objet': 'Test demande',
            'description': 'Description test',
            'organisme_id': self.organisme.id
        })
        
        self.assertTrue(demande.numero)
        self.assertEqual(demande.state, 'draft')
        self.assertTrue(demande.date_creation)
    
    def test_submit_demande(self):
        """Test de soumission d'une demande"""
        demande = self.Demande.create({
            'objet': 'Test demande',
            'description': 'Description test',
            'organisme_id': self.organisme.id
        })
        
        demande.action_submit()
        self.assertEqual(demande.state, 'submitted')
        self.assertTrue(demande.date_limite)
